# 09. 지급 볼트 교체와 통합

## 이 시나리오가 필요한 때

- 이자금 또는 상환금을 보관하는 지갑을 새로 바꿔야 할 때
- 키 분리, 보안사고 대응, 재무 분리를 해야 할 때

## 개념

- `RotatingVaultRouter`는 active vault를 바꿀 수 있습니다.
- 하지만 payout 시 이전 vault들도 같이 확인하므로, 과거 자산도 계속 청구 가능합니다.

## 운영 시나리오

1. 새 vault 생성
2. 새 vault를 active로 설정
3. 이후 적립금은 새 vault로 들어감
4. 기존 vault 잔액도 계속 지급 가능
5. 필요시 나중에 consolidate로 잔액을 모음

## 현재 프로젝트에서 직접 지원하는 부분

- 배포 스크립트는 기본 vault 생성까지 지원합니다.
- 운영 중 새 vault 생성/활성화/통합은 현재 계약 함수로 직접 호출합니다.

## 감사/운영 시 확인할 점

1. vault 교체 후에도 기존 시리즈의 claim이 끊기지 않는지
2. router가 payout 시 여러 vault를 순회해도 지급 정합성이 유지되는지
3. 운영팀이 어떤 자산이 어느 vault에 있는지 별도 장부를 갖는지

## 추천 운영 절차

1. 새 vault 생성 직후 소액 송금 테스트
2. 한 차수의 이자 청구를 실제로 수행
3. 기존 vault 잔액과 새 vault 잔액을 대조
4. 문제가 없을 때만 본격적으로 active 전환
